# PlanBridge UX 설계서 — 기획자·개발자 화면 상세

> 목표: 비개발자 현업도 5분 안에 첫 정책 등록, 개발자는 대시보드 열자마자 오늘 할 일 파악

---

## 1. 역할별 화면 구분

### 1.1 기획자(현업) — 보여야 할 것

```
사이드바 메뉴 (기획자 뷰)
──────────────────────────
🏠  홈 (내 활동 요약)
📋  내 변경 요청        ← 내가 요청한 것들의 진행 상태
📄  화면 기획서         ← 신규 화면 기획 작성
🔍  정책 검색           ← 기존 정책 찾아보기
⚙️  프로젝트 설정
```

**기획자가 절대 볼 필요 없는 것**: JSON 스키마, TypeScript 코드, API 엔드포인트, 컴포넌트 트리 구조

### 1.2 개발자 — 보여야 할 것

```
사이드바 메뉴 (개발자 뷰)
──────────────────────────
🏠  홈 (내 TODO 목록)   ← 열자마자 오늘 할 일
⚡  TODO 목록            ← 전체 할 일 목록
🔄  변경 요청            ← 구현할 요청들
🧩  컴포넌트             ← 컴포넌트별 정책 확인
📋  기획서               ← 신규 화면 기획서 확인
🔍  정책 검색
```

**개발자가 바로 써야 하는 것**: Claude Code 프롬프트 복사, TypeScript 코드 생성, 정책 JSON 확인

### 3.2 변경 요청 생성 (2단계 가이드)

```
─── Step 1: 어떤 화면의 무엇을 바꾸고 싶으신가요? ───────

  💡 팁: 브라우저에서 서비스 화면을 열고 익스텐션으로
         바꾸고 싶은 요소를 클릭하면 자동으로 연결됩니다.

  선택된 요소:
  ┌─────────────────────────────────────────────────┐
  │  🔲 상품 관리 > 할인가 입력 필드                │
  │     (클릭해서 다시 선택할 수 있어요)             │
  └─────────────────────────────────────────────────┘

  지금 어떻게 되어있나요?
  ┌─────────────────────────────────────────────────┐
  │  할인율이 최대 90%까지만 가능한데,              │
  │  시즌 특가 때 95%까지 필요합니다.               │
  └─────────────────────────────────────────────────┘

  [다음 →]

─── Step 2: 어떻게 바꾸고 싶으신가요? ──────────────────

  어떻게 바뀌었으면 하나요?
  ┌─────────────────────────────────────────────────┐
  │  95%까지 할인 가능하게 해주세요.               │
  │  단, 90% 초과는 슈퍼관리자만 가능하고           │
  │  사유를 반드시 입력하도록 해주세요.             │
  └─────────────────────────────────────────────────┘

  중요도:
  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
  │  낮음    │ │ 보통 ✓  │ │  높음    │ │  긴급    │
  └──────────┘ └──────────┘ └──────────┘ └──────────┘

  [제출하기] → AI가 자동으로 개발 TODO를 만들어드려요
```

  ✓ 키보드 단축키 지원 (C: 복사, D: 완료, N: 다음)
  ✓ 코드 블록은 항상 복사 버튼 포함
